## [ JSON ]

### 핵심 한 줄 요약
    JSON이 익숙하다고 해서 막 쓰지 말고, 주의점과 단점을 명심하여 작성하자.

### 인트로
- JSON 은 숫자, 문자, 불리언 등 여러 형태의 데이터를 키와 값으로 구조화된 객체에 담아 처리하는 규칙이다.
- 텍스트 기반이라 데이터를 읽기 쉽고, 가공도 편하다.

### 특징
- 문자열 인코딩:
  - UTF-8 인코딩만 허용한다.
- 주석:
  - 주석을 지원하지 않는다.
- 구조:
  - 너무 익숙해서 단어만 들어도 떠오른다.

### 주의점
- 키의 값으로 "null" 은 사용하지 않는게 좋다.
- 키 이름에 대한 표준이나 관례가 없어서 같이 쓰는 규칙을 잘 정하고 통일해야 한다.

### 단점
- 데이터를 읽기는 쉽지만 표현하는데 드는 비용이 크다.
- 클라이언트, 서버 간에 싱크를 잘 맞춰야 한다. => 이게 은근 어렵다.
- 짧은 시간안에 많은 데이터를 주고받아야 하는 상황에서는 비효율적이다.
