## [ RESTful API ]

### 핵심 한 줄 요약
    작성한 내용들은 아주 기본이 되는 내용이다. 기본에 충실하여 RESTful 하게 API를 만들도록 노력하자.

### 인트로
- RESTful API는 서버와 클라이언트가 메세지를 주고받을 때 가장 많이 사용하는 통신 규격이다.
- REST: 
  - representational state transfer 의 약자로 분산 시스템을 위한 소프트웨어의 아키텍처의 한 형태를 가리킨다.
  - RESTful 하다는 건 REST의 조건을 만족한다는 뜻이다.
  - 요청 주소와, 메서드, JSON 규격을 이용하여 API를 정의하고 사람이 읽기 쉬운 형태이기 때문에 가장 범용적으로 사용한다.

### 요청 주소
- "http://localshot:5000/v1/posts" 처럼 버전을 명시하는게 좋다.
- 호환성이 깨지면 새로운 버전을 만들어 사용한 다음 충분한 기간을 둘 다 사용하게 하는 것이 좋다.
- 그리고 나서 전 버전의 지원종료 사실을 알리고 새 버전으로 넘어갈 시간을 충분히 주는게 좋다.

### 메서드 간략 의미
- GET: 읽기, 조회
- POST: 생성, 쓰기
- PUT/PATCH: 업데이트
- DELETE: 삭제
- * 메서드 대신에 요청 주소에 행동을 정의하는 경우가 있는데 좋은 디자인은 아니다.

### 메시지 바디와 쿼리 스트링
- POST 메서드를 사용하면 일반적으로 메시지 바디에 JSON을 담아 전달한다.
- 쿼리 스트링:
  - URL 주소에 ? 문자랑 & 문자가 예약어라 두 문자를 인수 값으로 사용할 수 없다.
  - URL 주소는 아스키 코드 값만 허용하기 떄문에 Base64로 변환해야 한다.
  - 근데 Base64에도 URL 주소에 허용 안되는 값이 있기 때문에 URL-safe 인코딩을 추가로 해줘야 한다.
- 메시지 바디에 담을 때는 별다른 처리가 필요없지만, 바디를 사용할 수 없는 상황에서는 반드시 인코딩된 쿼리 스트링을 사용해야 한다.
